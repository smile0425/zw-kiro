## 二、金字塔原理：PPT 的核心方法论

每份 PPT 必须严格遵循金字塔原理，确保逻辑层层递进：

### 结构框架（自上而下）

```
核心主张（1 句话概括整个 PPT 要传递的信息）
+-- 支撑论点 1（为什么？/ 是什么？）
|   +-- 论据 / 数据 / 案例
|   +-- 论据 / 数据 / 案例
+-- 支撑论点 2
|   +-- 论据 / 数据 / 案例
|   +-- 论据 / 数据 / 案例
+-- 支撑论点 3
|   +-- 论据 / 数据 / 案例
|   +-- 论据 / 数据 / 案例
+-- 行动号召（CTA）
```

### 逻辑递进原则

1. **MECE 原则**：每一层的论点之间相互独立、完全穷尽，不重叠不遗漏
2. **归纳与演绎**：优先使用归纳法（从具体案例到一般结论），让听众自己得出结论
3. **SCQA 开场**：Situation（背景）- Complication（冲突）- Question（问题）- Answer（答案）
4. **三点法则**：每个层级的论点控制在 3-5 个，不超过 5 个
## 三、PPT 页面设计规范

### 标准页面结构（每页 PPT）

```
页面标题：一句话结论（不是主题词，而是观点）
副标题/说明：补充说明（可选）
正文内容：3-5 个要点，每个要点一句话
数据/图表：用数字说话
```

### 页面类型

| 类型 | 用途 | 内容要求 |
|------|------|----------|
| 封面页 | 开场 | 主题、副标题、演讲者、日期 |
| 目录页 | 导航 | 3-5 个章节标题 |
| 观点页 | 传递核心信息 | 标题即结论 + 3 个支撑点 |
| 数据页 | 用数据说服 | 1 个核心数据 + 对比/趋势 |
| 案例页 | 增强可信度 | 合作伙伴名称 + 具体成果 |
| 对比页 | 突出优势 | 竞品对比或前后对比 |
| 时间线页 | 展示历程 | 关键里程碑，不超过 6 个节点 |
| 架构页 | 展示技术全景 | 分层架构图描述 |
| CTA 页 | 行动号召 | 明确的下一步 + 联系方式 |
| 结尾页 | 收尾 | 金句 + 感谢 |

### 每页文案要求

- **标题**：必须是一个完整的观点句，不是主题词。错误示例："openvela 生态"；正确示例："openvela 已汇聚 100+ 合作伙伴，形成完整产业链"
- **正文**：每个要点不超过 2 行，使用短句
- **数据**：关键数据要放大加粗，成为视觉焦点
- **过渡**：每页之间有逻辑衔接，不跳跃

## 五、工作流程

当用户提供飞书文档链接或长文内容时，按以下步骤执行：

### 第一步：内容提炼
1. 获取并阅读完整文档内容
2. 识别核心主张、关键论点、重要数据、典型案例
3. 按金字塔原理梳理逻辑层次

### 第二步：确定 PPT 框架
1. 确定核心主张（一句话）
2. 拆解为 3-5 个支撑论点
3. 为每个论点匹配数据和案例
4. 设计 SCQA 开场
5. 规划页面数量（通常 15-25 页）

### 第三步：逐页输出
为每一页 PPT 输出：
- 页码和页面类型
- 页面标题（观点句）
- 正文要点（3-5 个）
- 演讲者备注（口头补充说明）
- 视觉建议（图表类型、配图方向）

### 第四步：检查清单
- [ ] 金字塔结构：是否自上而下、层层递进？
- [ ] MECE：每层论点是否不重叠不遗漏？
- [ ] 标题即结论：每页标题是否是完整观点句？
- [ ] 数据支撑：关键论点是否有数据佐证？
- [ ] 逻辑过渡：页面之间是否有衔接？
- [ ] 品牌规范：openvela 是否全部小写？
- [ ] 受众适配：语言是否适合目标受众？
- [ ] 行动号召：结尾是否有明确的 CTA？
- [ ] 时间控制：页数是否适合演讲时长？（约 1-2 分钟/页）
